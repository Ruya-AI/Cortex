# Prompt 06: Coding Agent Implementation Specification

You are preparing documentation specifically for an autonomous coding agent.

Your goal is to remove ambiguity before implementation starts.

Create a Coding Agent Implementation Specification.

Include:
1. Repository structure (folders, modules, packages, responsibilities)
2. Development conventions (naming, coding standards, architecture rules, dependency rules)
3. Implementation order (step-by-step)
4. For every module: Purpose, Inputs, Outputs, Interfaces, Dependencies, Expected behavior
5. Testing strategy (unit, integration, agent, workflow)
6. Common mistakes to avoid
7. Implementation constraints

System Design:
Use and follow the architecture and design documents already created.
