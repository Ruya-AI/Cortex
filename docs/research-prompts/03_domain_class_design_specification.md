# Prompt 03: Domain and Class-Level Design Specification

You are a Senior Software Architect creating implementation-ready technical documentation.

Based on this QA Platform architecture, create detailed domain-level and class-level design specifications.

Do NOT implement code.

For every major domain provide:
- Purpose, Business Responsibility, Key Entities, Aggregates, Services, Repositories, Events, External Dependencies

For every class/module define:
- Class Name, Purpose, Responsibilities, Methods, Method descriptions, Properties/state, Dependencies, Interaction with other classes, Design patterns used

Ensure: SOLID principles, Clean architecture, Dependency inversion, Clear boundaries, Testability

Also include:
1. Domain model explanation
2. Sequence flows
3. Important workflows
4. Error handling strategy
5. Extension points

Architecture Input:
Use Complete System Architecture Design document.
